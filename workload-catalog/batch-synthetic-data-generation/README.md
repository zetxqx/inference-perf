# Batch Synthetic Data Generation

This workload simulates the generation of large volumes of synthetic data (e.g., training data, test cases) using a language model.

## 1. Use Case and Distributions
**What it is**: The system acts as a high-throughput data engine. It is specifically prompted to generate diverse training examples, synthetic chat turns, or code snippets.
**Why the distributions**:
- **Input Sequence Length (ISL)**: Log-Normal. Most inputs are tiny, highly parameterized seed prompts (e.g., *"Generate 50 JSON records of... "*). The long right tail accounts for workflows where an entire source document is injected to be synthetically rewritten or augmented.
- **Output Sequence Length (OSL)**: Uniformly High. The model is prompted to hit maximum context limits to generate as many examples as possible in a single pass.

## 2. Reference Datasets
- **[Cosmopedia](https://huggingface.co/datasets/HuggingFaceTB/cosmopedia)**: A large dataset of synthetic textbooks, stories, and blog posts generated by Mixtral, useful for training or evaluating on synthetic data.
- **[The Stack v2](https://huggingface.co/datasets/bigcode/the-stack-v2)**: A large dataset of source code, useful as a seed or reference for generating synthetic code data.

## 3. System Impact
- **High Throughput Demand**: Puts extreme pressure on the decoding phase as it generates large outputs continuously.
- **KV Cache Pressure**: Large outputs mean the KV cache grows significantly, requiring efficient management or offloading.
- **Fault Tolerance**: Long-running batch jobs benefit from fault-tolerant queueing to handle failures without losing progress.
